---
title: Edit Widgets Catalog
description: Complete reference of widgets available for data input in create and edit forms.
---

Edit widgets are the form inputs operators use when creating or editing records. Each field can have a different edit widget depending on the type of input you want to collect.

To change an edit widget, enter [Layout Editor](/product/build/layout-editor) mode, switch to "New record" or "Edit record" view, click a field, and select a widget from the configuration panel.

---

## Text Widgets

### Text Input
The default widget for short string fields. A single-line input. Best for names, identifiers, short descriptions.

### Textarea
A multi-line text input for longer content. Configure the number of visible rows. Best for notes, comments, and descriptions.

### Rich Text Editor
A WYSIWYG editor with formatting controls (bold, italic, lists, headings, links). Stores content as HTML. Best for formatted descriptions, email templates, and long-form content.

### Code Editor
A syntax-highlighted code editor with line numbers. Select the language mode (JavaScript, SQL, JSON, Python, etc.) for appropriate highlighting. Best for configuration values, scripts, and queries stored in the database.

### Email Input
A text input with email format validation. Operators see an error if the entered value isn't a valid email address.

### URL Input
A text input with URL format validation. Operators see an error if the entered value isn't a valid URL.

### Phone Input
A text input with phone number formatting. Displays a country code selector. Best for international phone number fields.

### Password Input
A masked text input — characters appear as dots. Used for fields that store credentials or sensitive tokens. Value is never pre-filled in edit mode.

---

## Number Widgets

### Number Input
The default for numeric fields. Shows increment/decrement buttons. Configure min, max, and step values to constrain input.

### Currency Input
A number input with a currency symbol prefix. Configure the currency code and locale. Validates that the value is a valid monetary amount.

### Slider
A draggable slider for selecting a value within a defined range. Good for scores, ratings, percentages, and any bounded numeric input.

### Stepper
Increment/decrement buttons without a text input. Best for small integer values where precision isn't needed (quantity, priority level, etc.).

---

## Selection Widgets

### Dropdown
A select menu with predefined options. Best for fields with a fixed set of values (status, category, type). Operators can only pick from the list.

Options can be:
- **Static** — defined manually in the widget configuration
- **Dynamic** — loaded from another collection via a Smart Action

### Radio Buttons
Shows all options at once as radio buttons. Best when there are 2–5 options and you want them all visible without clicking. Operators can select one option.

### Checkbox (edit)
A single checkbox for boolean fields. The operator toggles it on or off.

### Multi-select
Like Dropdown but allows selecting multiple values. Stores as an array. Best for tags, categories, and labels where multiple values apply simultaneously.

### Autocomplete
A text input that searches and filters options as you type. Best for large option lists where scrolling a dropdown would be impractical. Options can be static or dynamically fetched.

### Tags Input
Lets operators type values and press Enter to create tags. Values are stored as an array of strings. No predefined list required — operators create tags freely.

---

## Date & Time Widgets

### Date Picker
A calendar popup for selecting a date. Stores the value as a date without time component. Supports min/max date constraints.

### DateTime Picker
Combines a calendar with a time selector. For fields that store full timestamps. Configure timezone handling.

### Time Picker
A time-only input (hours and minutes, optionally seconds). For time-of-day fields not associated with a specific date.

### Date Range Picker
Two date pickers for selecting a start and end date. Stores two values. Best used in filters and Smart Action forms.

---

## Boolean Widgets

### Checkbox
A simple checkbox — checked means `true`. The most common widget for boolean fields.

### Toggle Switch
A toggle that switches between on and off states. More prominent than a checkbox. Good for settings, feature flags, and enable/disable states.

---

## Media Widgets

### File Uploader
A file upload input. Operators can browse for a file or drag and drop it. Configure accepted file types and max file size. Uploads the file and stores the resulting URL in the field.

### Image Uploader
Like File Uploader but restricted to image types, with a preview of the selected image before saving. Supports cropping if configured.

### Drag & Drop Uploader
A large drop zone that accepts one or multiple files. Better UX than a button when file upload is the primary action on a form.

---

## Relationship Widgets

### BelongsTo Selector
A searchable dropdown that lets operators pick a related record. Searches by the related collection's summary field. Creates the association when the record is saved.

### HasMany Editor
Displays existing related records in a list, with options to add new records or remove existing associations. Best for managing one-to-many relationships inline.

### Search & Select
An autocomplete input that searches across a related collection as the operator types. More powerful than the BelongsTo Selector for large related collections.

---

## Special Widgets

### JSON Editor
A code editor pre-configured for JSON, with syntax validation. Shows an error if the entered value is not valid JSON. Best for configuration objects and metadata fields.

### Color Picker
A color selector with a swatch palette and hex input. Stores the value as a hex code (`#FF5733`) or RGB string.

### Rating Input
Star rating input where operators click to set a value. Configure the max number of stars and whether half-stars are allowed.

### Address Input
A structured address form with separate inputs for street, city, zip code, country, etc. Stores as a structured object or a formatted string.

### Location Picker
An embedded map that lets operators pin a location. Stores latitude and longitude coordinates. Useful for delivery addresses, store locations, and event venues.

---

## Validation

Most edit widgets support built-in validation options:

| Validation | Description |
|------------|-------------|
| **Required** | Field must be filled before saving |
| **Min / Max** | For numbers: value bounds. For text: character length limits |
| **Pattern** | Regex validation for custom formats |
| **Custom error message** | Override the default validation error text |

Validation runs client-side for immediate feedback, and can also be enforced server-side in Smart Action hooks.

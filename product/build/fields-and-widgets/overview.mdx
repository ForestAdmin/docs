Every piece of data in Forest Admin is represented by a **field**. Every field is rendered using a **widget**. Understanding this distinction helps you configure exactly how your data looks and behaves.

## Fields vs. Widgets

A **field** is a data definition — it comes from your database schema or is computed by your agent. Fields have a name, a type (`String`, `Number`, `Date`, `Boolean`, etc.), and a value.

A **widget** is the UI component used to display or edit that field. The same field can be rendered differently depending on where it appears and what you want operators to do with it.

For example, a `String` field named `status` could be displayed as:
- A plain text label in the table view
- A colored **status badge** in the detail view
- A **dropdown selector** in the edit form

## Display vs. Edit Widgets

Widgets come in two categories:

| Category | When it's used | Purpose |
|----------|---------------|---------|
| **Display widget** | Table view, detail view, Workspaces | Shows data in a readable format |
| **Edit widget** | Create form, edit form, inline editing | Accepts input from operators |

A field can have different widgets for display and edit. A date field might display as a relative time ("3 days ago") but edit with a date picker calendar.

## Widget Categories

### Text

For short and long text, emails, URLs, code, and rich content.

Examples: Plain text, Textarea, Markdown renderer, Code block, Email link, URL link

### Number

For integers, decimals, percentages, currencies, and durations.

Examples: Number, Currency (with symbol), Percentage, Progress bar

### Date & Time

For timestamps, dates, times, and relative time.

Examples: Date, DateTime, Time, Relative time ("2 hours ago")

### Boolean

For true/false values.

Examples: Checkbox, Toggle, Badge (Yes/No), Icon (✓/✗)

### Media & Files

For images, documents, videos, and binary data.

Examples: Image preview, File download link, Video player, Audio player

### Relationships

For BelongsTo and HasMany associations.

Examples: Related record link, Record count, Search & select picker

### Special

For structured or complex data.

Examples: JSON viewer, JSON editor, Color swatch, Star rating, Map/location

## Configuring Widgets

To change a widget for a field:

1. Enter **Layout Editor** mode
2. Navigate to the view you want to change (table, detail, or form)
3. Click on a field to open its configuration panel
4. Select a new widget from the **Widget** dropdown
5. Adjust widget-specific options (format, color, validation, etc.)
6. Save your changes

<Info>
  Widget availability depends on the field's data type. A Boolean field can't use a text input widget, for example.
</Info>

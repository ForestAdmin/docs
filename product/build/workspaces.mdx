A **Workspace** is a custom interface you build inside Forest Admin, tailored to a specific use case. Instead of navigating between multiple collections, operators get a single focused view that shows exactly what they need and lets them take action immediately.

Think of a Workspace as the difference between a generic spreadsheet and a purpose-built tool: same data, dramatically better experience.

## Why use workspaces

The default "Data" tab gives operators access to all your collections, but it's generic. Workspaces let you:

- **Focus** - show only the data relevant to a specific task
- **Contextualize** - display related information side by side without navigating away
- **Act faster** - put action buttons directly next to the data that triggers them
- **Guide** - add text, sections, and structure to guide users through it

Workspaces are built with a **drag-and-drop editor**, no code required for most use cases.

## Workspace components

A Workspace is composed of **components**, broken down into 5 categories, that you drag onto a canvas and configure:

**Layout & Guidance** components aim to design your workspace in an intuitive way, and guide end users through it:

| Component | Purpose |
| --- | --- |
| **Text** | Headers, instructions, labels with optional dynamic templating |
| **Section** | Groups related components within a section |
| **Tabs** | Tabbed section to segment groups of components |
| **Link** | A link to a record, external URL, or another workspace |
| **Dividers** | Visual separators |

**Source** components set the context of your workspace, for all data-related components to plug into:

| Component | Purpose |
| --- | --- |
| **Search** | Lets users search for a record from a collection |
| **Collection** | Lets users select a record from a list, optionally filtered |
| **Inbox** | Picks the highest priority record for the user to process |
| **Dropdown** | A selector that filters connected components dynamically |
| **Date Picker** | A date picker that filters connected components dynamically |
| **Toggle** | A boolean switch that filters connected components dynamically |
| **Input** | Free text input that filters connected components dynamically |

**Data & Action** components provide users with data, actions, and workflows available on the record coming from a Source component:

| Component | Purpose |
| --- | --- |
| **Field** | Shows the value of a specific field from a selected record |
| **Action** | Triggers an Action on a selected record |
| **Workflow** | Triggers or resumes a Workflow on a selected record |

**Analytics** components provide charts and dashboards for extra context:

| Component | Purpose |
| --- | --- |
| **Chart** | Embeds a chart or metric (single value, distribution, etc.) |
| **Metabase** | Embeds a Metabase dashboard using an iframe |

The **Custom** components lets you write your own, based on your own business logic and preferences in terms of visual representation.

## Templating: making workspaces dynamic

The real power of Workspaces comes from **Templating** — the ability to reference values from other components dynamically. Click on the `{}` button in any text, filter, or link to see available variables.

For example:

- A **Text** component can display `Welcome, {{currentUser.name}}`
- A **Collection** component can filter records where `company_id = {{collection1.selectedRecord.id}}`
- A **Link** can redirect to `https://app.example.com/users/{{collection1.selectedRecord.id}}`

When an operator selects a row in `collection1`, all connected components update instantly.

**Step 1 — Create a new workspace**

Click on `+ Add New` in the left sidebar, select **Workspace**, and give it a descriptive name that reflects the use case (e.g. "Customer Onboarding", "CRM").

**Step 2 — Add a Collection component**

Drag a **Collection** from the sidebar onto the canvas. Select which collection to display and optionally apply a segment (e.g. "Pending applications"). Set the row click behavior to **"Select a record"** to enable dynamic updates.

**Step 3 — Add context with Field components**

Drag **Field** components onto the canvas. Link them to your Collection and select which fields to display. When an operator clicks a row, these fields update to show the selected record's values.

**Step 4 — Add Actions**

Drag an **Action** and link it to your Collection. Now operators can click a row and immediately trigger the right action — no need to open the record detail.

**Step 5 — Exit the builder and test it**

Click "Exit builder" to switch to the operator view. Test the workflow by clicking rows and triggering actions.

## Sharing workspaces

Workspaces are available to all operators in your project by default. Control access using [Roles & Permissions](/product/control/roles-permissions), you can limit Workspace visibility to specific teams.

Operators can also share a Workspace URL with a pre-applied filter state, making it easy to hand off specific work items.

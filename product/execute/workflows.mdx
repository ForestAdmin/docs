**Workflows** in Forest Admin allow operators to trigger and monitor automated multi-step processes directly from the admin panel — without needing to run scripts or contact developers.

## What are Workflows?

A Workflow chains together multiple operations into a single automated sequence. Examples include:

- Sending a welcome email → creating an account → notifying the sales team
- Validating a document → updating a status → triggering an external sync
- Running a nightly report → distributing it to stakeholders

## Executing workflows

Workflows can be executed from the List View (on a single record), the Summary or Details view of a record, or from a Workspace.

### AI-powered steps

The **Get Data**, **Update Data**, **Trigger Action**, and **MCP Task** steps aim to provide end users with the tools they need to efficiently complete actions in Forest Admin and 3rd party apps.

Each time one of these steps is completed, the workflow context gathers more information, increasing the AI's ability to help with subsequent steps — for example by pre-filling fields when updating data, or composing an email with the relevant context when notifying a customer via a tool like Zendesk.

Every AI-powered step provides a **Handle Manually** option, allowing users to bypass the suggestion if it is not relevant or if it encountered an error. Subsequent steps would lack that context, which may affect the rest of the workflow.

### Automated steps

Some steps in a Workflow may be automated, meaning they won't require any user input to complete, namely for the AI-powered steps listed above, and some Decisions.

In some cases, for example if Forest's AI is lacking context to make a decision, or a 3rd party system is not responding, the user may be required to take over and complete the step manually, similarly to the Handle Manually option.

### Saving & resuming

During execution, a workflow can be closed at any time, prompting the user to either **Save** it or **Abort** it.

Saved workflows can be picked up by another user, after any length of time. They resume automatically when triggered from a record's Summary View or a Workspace, and can be resumed from the **To Continue** section when accessed from the List or Record View.

### Revising steps

Workflows help reduce mistakes in complex processes, but errors may still happen. If you need to correct something, you can always go back and revise a completed step.

After completing a step, click the **Revise** button to redo it. Since workflows can branch based on your choices, revising a step will cancel any steps completed after it — ensuring the workflow follows the correct path based on your updated input.

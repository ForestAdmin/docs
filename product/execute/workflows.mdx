**Workflows** in Forest Admin allow operators to trigger and monitor automated multi-step processes directly from the admin panel — without needing to run scripts or contact developers.

## What are Workflows?

A Workflow chains together multiple operations into a single automated sequence. Examples include:

- Sending a welcome email → creating an account → notifying the sales team
- Validating a document → updating a status → triggering an external sync
- Running a nightly report → distributing it to stakeholders

Workflows can be triggered **manually** by operators, **automatically** based on events, or **on a schedule**.

## Triggering a Workflow

Depending on how your workflows are configured, you can trigger them from:

- **A record detail view** — a workflow button appears alongside standard actions.
- **A collection action** — workflows can be exposed as global or bulk actions.
- **A scheduled trigger** — workflows that run automatically don't require manual triggering.

To trigger a workflow manually:

1. Navigate to the relevant record or collection.
2. Click the workflow trigger button or select it from the **Actions** menu.
3. Fill in any required input form and click **Start**.

## Monitoring Workflow Progress

Once triggered, you can monitor the workflow's execution:

- **Status indicator** — shows whether the workflow is pending, running, completed, or failed.
- **Step-by-step progress** — view which step is currently executing.
- **Activity log** — each workflow execution is recorded in the [Activity Logs](/product/manage/activity).

## Workflow Results

When a workflow completes, the result is shown:

| Result | Description |
|--------|-------------|
| **Completed** | All steps executed successfully |
| **Failed** | One or more steps encountered an error |
| **Cancelled** | The workflow was manually stopped or timed out |

Failed workflows show which step failed and the error message. Depending on the configuration, some workflows support retrying from the failed step.

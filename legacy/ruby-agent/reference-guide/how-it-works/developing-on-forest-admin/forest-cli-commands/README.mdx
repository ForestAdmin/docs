# Forest CLI commands

You've seen in the previous section how [developing on Forest Admin](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/overview) leverages our powerful **Forest CLI** to manage your UI changes.

### Installing Forest CLI

To install Forest CLI, run the following command in your terminal:

```
npm install -g forest-cli
```

For further details on the package, check out [this page](https://www.npmjs.com/package/forest-cli).

### Using Forest CLI

In the following pages, we'll cover all available Forest CLI commands in details, from introduction to advanced usage. For now, there are 6 commands:

- [login](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/login)
- [init](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/init)
- [branch](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/branch)
- [switch](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/switch)
- [set-origin](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/set-origin)
- [push](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/push)
- [environments:reset](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/environments-reset)
- [environments:create](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/environments-create)
- [deploy](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/deploy)
- [schema:diff \[beta\]](/legacy/ruby-agent/reference-guide/how-it-works/developing-on-forest-admin/forest-cli-commands/schema-diff-beta) &#x20;

Some additional commands might be added in the future. In the meantime, those should be largely sufficient to manage your development workflow.

<Info>
Be aware that almost all commands take the `FOREST_ENV_SECRET` env variable, provided on the command or inside your _.env_ file, to know on which environment the command is run.
</Info>

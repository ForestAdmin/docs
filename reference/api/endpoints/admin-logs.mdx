# Admin Logs API

The Admin Logs API provides access to administrative operation records, enabling you to track configuration changes, permission updates, and other admin panel activities for security and compliance purposes.

## Overview

Admin logs capture administrative actions performed in your Forest Admin project, including:

- **Configuration Changes** - Updates to collections, fields, layouts
- **Permission Management** - Role and access control modifications
- **Team Management** - User invitations, role assignments, removals
- **Integration Changes** - API key generation, webhook configuration
- **Environment Settings** - Project settings and environment variables

## Endpoints

### GET /api/v1/admin-logs

Retrieve a paginated list of admin logs with optional filtering.

<ParamField header="Authorization" type="string" required>
  Bearer token for API authentication. Format: `Bearer YOUR_API_TOKEN`
</ParamField>

#### Query Parameters

<ParamField query="start_date" type="string">
  Filter logs from this date (ISO 8601 format: `YYYY-MM-DD` or `YYYY-MM-DDTHH:mm:ss.sssZ`)

  **Example:** `2024-01-01` or `2024-01-01T00:00:00.000Z`
</ParamField>

<ParamField query="end_date" type="string">
  Filter logs up to this date (ISO 8601 format: `YYYY-MM-DD` or `YYYY-MM-DDTHH:mm:ss.sssZ`)

  **Example:** `2024-01-31` or `2024-01-31T23:59:59.999Z`
</ParamField>

<ParamField query="admin_id" type="string">
  Filter logs by specific admin user ID

  **Example:** `admin_123`
</ParamField>

<ParamField query="operation_type" type="string">
  Filter by operation type

  **Options:** `config_change`, `permission_update`, `team_management`, `integration_change`, `settings_update`
</ParamField>

<ParamField query="severity" type="string">
  Filter by severity level

  **Options:** `info`, `warning`, `critical`
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination (starts at 1)
</ParamField>

<ParamField query="limit" type="integer" default="100">
  Number of results per page (max: 1000)
</ParamField>

<ParamField query="sort" type="string" default="-created_at">
  Sort order for results

  **Options:** `created_at`, `-created_at` (descending)
</ParamField>

## Use Cases

### Security Auditing

Monitor critical permission changes:

```javascript
// Track all critical operations
const criticalLogs = await fetch(
  'https://api.forestadmin.com/api/v1/admin-logs?severity=critical&limit=100',
  {
    headers: { 'Authorization': `Bearer ${apiToken}` }
  }
);

const data = await criticalLogs.json();

// Alert security team
if (data.data.some(log => log.attributes.operation_type === 'permission_update')) {
  sendSecurityAlert('Critical permission change detected');
}
```

### Compliance Reporting

Generate compliance reports for administrative activities:

```javascript
// Export admin logs for quarterly audit
const quarterStart = '2024-01-01';
const quarterEnd = '2024-03-31';

const response = await fetch(
  `https://api.forestadmin.com/api/v1/admin-logs?start_date=${quarterStart}&end_date=${quarterEnd}&limit=1000`,
  {
    headers: { 'Authorization': `Bearer ${apiToken}` }
  }
);

const logs = await response.json();

// Generate compliance report
generateComplianceReport(logs.data, 'Q1 2024');
```

### Team Activity Monitoring

Track team management operations:

```javascript
// Monitor team changes
const teamLogs = await fetch(
  'https://api.forestadmin.com/api/v1/admin-logs?operation_type=team_management&limit=50',
  {
    headers: { 'Authorization': `Bearer ${apiToken}` }
  }
);

const data = await teamLogs.json();

// Analyze team growth
const additions = data.data.filter(log =>
  log.attributes.operation_name === 'Add team member'
);
const removals = data.data.filter(log =>
  log.attributes.operation_name === 'Remove team member'
);

console.log(`Team additions: ${additions.length}`);
console.log(`Team removals: ${removals.length}`);
```

### Configuration Change Tracking

Monitor and rollback configuration changes:

```javascript
// Track recent config changes
const configLogs = await fetch(
  'https://api.forestadmin.com/api/v1/admin-logs?operation_type=config_change&limit=20',
  {
    headers: { 'Authorization': `Bearer ${apiToken}` }
  }
);

const data = await configLogs.json();

// Find specific change to rollback
const layoutChange = data.data.find(log =>
  log.attributes.operation_name === 'Update collection layout' &&
  log.attributes.target.name === 'users'
);

if (layoutChange) {
  // Rollback to previous state
  const previousState = layoutChange.attributes.changes.before;
  rollbackConfiguration('users', 'layout', previousState);
}
```

### Integration Change Monitoring

Track API key and webhook changes:

```javascript
// Monitor integration security
const integrationLogs = await fetch(
  'https://api.forestadmin.com/api/v1/admin-logs?operation_type=integration_change&limit=50',
  {
    headers: { 'Authorization': `Bearer ${apiToken}` }
  }
);

const data = await integrationLogs.json();

// Alert on new API key generation
const apiKeyChanges = data.data.filter(log =>
  log.attributes.target.type === 'api_key'
);

if (apiKeyChanges.length > 0) {
  notifySecurityTeam('New API keys generated', apiKeyChanges);
}
```

## Best Practices

### 1. Monitor Critical Operations

Focus on high-severity operations:

```javascript
// Good - monitor critical changes
const criticalLogs = await getAdminLogs({
  severity: 'critical',
  start_date: new Date(Date.now() - 24*60*60*1000).toISOString()
});

// Set up real-time alerts
criticalLogs.forEach(log => {
  if (log.attributes.operation_type === 'permission_update') {
    alertSecurityTeam(log);
  }
});
```

### 2. Regular Compliance Exports

Schedule regular exports for compliance:

```javascript
// Daily export of admin logs
async function dailyComplianceExport() {
  const yesterday = new Date(Date.now() - 24*60*60*1000)
    .toISOString().split('T')[0];
  const today = new Date().toISOString().split('T')[0];

  const logs = await getAdminLogs({
    start_date: yesterday,
    end_date: today,
    limit: 1000
  });

  // Store in compliance archive
  await complianceArchive.store(logs, yesterday);
}

// Run daily at midnight
schedule.daily('00:00', dailyComplianceExport);
```

### 3. Detect Anomalies

Implement anomaly detection:

```javascript
async function detectAnomalies() {
  const recentLogs = await getAdminLogs({
    start_date: new Date(Date.now() - 60*60*1000).toISOString(), // Last hour
    limit: 1000
  });

  // Group by admin
  const activityByAdmin = groupBy(recentLogs, log => log.attributes.admin.id);

  // Alert on unusual activity patterns
  Object.entries(activityByAdmin).forEach(([adminId, logs]) => {
    if (logs.length > 50) {
      alertSecurityTeam(`Unusual activity from admin ${adminId}: ${logs.length} operations in 1 hour`);
    }
  });
}
```

### 4. Track Environment Changes

Separate production from other environments:

```javascript
// Monitor production changes separately
const productionLogs = await getAdminLogs({
  environment: 'production',
  severity: 'critical'
});

// Require approval for critical production changes
productionLogs.forEach(log => {
  if (!hasApproval(log)) {
    flagForReview(log);
  }
});
```

### 5. Document Change History

Maintain detailed change documentation:

```javascript
async function documentChanges(startDate, endDate) {
  const logs = await getAdminLogs({
    start_date: startDate,
    end_date: endDate,
    limit: 1000
  });

  const documentation = logs.map(log => ({
    timestamp: log.attributes.created_at,
    admin: log.attributes.admin.name,
    operation: log.attributes.operation_name,
    target: log.attributes.target.name,
    severity: log.attributes.severity,
    changes: log.attributes.changes
  }));

  // Generate change log document
  generateChangeLog(documentation, startDate, endDate);
}
```

## Rate Limits

Admin Logs API is subject to standard rate limits:

- **60 requests per minute** per API token
- **1,000 requests per hour** per API token
- **10,000 requests per day** per API token

See [Rate Limits](/reference/api/rate-limits) for detailed information on handling rate limits.

# forest login

Sign in to your Forest Admin account to enable CLI commands.

## Usage

```bash
forest login [options]
```

## Options

| Option | Description |
|---|---|
| `-e, --email` | Your Forest Admin account email |
| `-P, --password` | Your Forest Admin account password (ignored if `--token` is set) |
| `-t, --token` | Your Forest Admin account token (replaces password) |

## Authentication methods

### Interactive login

Running `forest login` without arguments will prompt you for your email and password:

```bash
$ forest login
? Enter your email: you@company.com
? Enter your password: ********
✅ You are now logged in.
```

### Login with email and password

```bash
forest login --email you@company.com --password yourpassword
```

### Login with an application token

Application tokens are a secure alternative to passwords, especially useful in CI/CD pipelines or shared environments.

```bash
forest login --token YOUR_APPLICATION_TOKEN
```

To create an application token, go to your **Account settings** (accessible from the top-right dropdown in the Forest Admin UI) and generate a new token.

<Tip>
Use application tokens in automated environments to avoid storing your password in plain text.
</Tip>

## Logging out

To sign out of your account:

```bash
forest logout
```

## Checking the current user

To display which account is currently authenticated:

```bash
forest user
```

## Token storage

The CLI stores your session token locally in `~/.forest.d/`. This directory is created automatically on first login.

<Warning>
Do not commit the `~/.forest.d/` directory to version control. It contains your authentication credentials.
</Warning>

## Troubleshooting

**Authentication fails with correct credentials**
Check that your account exists and that you're using the correct email. If you log in via SSO, use an application token instead.

**Token expired**
Run `forest login` again to obtain a fresh session.

**Command requires authentication**
Most CLI commands require you to be logged in. Run `forest login` before using other commands.

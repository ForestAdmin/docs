---
title: Deploy to AWS
description: "Step-by-step guide to deploy the Forest Admin agent to Amazon Web Services using EC2, ELB, ACM and Route53"
---

This guide walks you through deploying your Forest Admin agent to AWS using EC2, an Application Load Balancer, ACM for SSL, and Route53 for DNS.

<Info>
  You'll need an AWS account. [Sign up here](https://aws.amazon.com/) if you don't have one.
</Info>

## Step 1: Launch an EC2 instance

1. Navigate to the EC2 dashboard and click **Launch Instance**
2. Choose an AMI such as **Amazon Linux 2023 AMI**
3. Select **t2.micro** (part of the AWS Free Tier)
4. Select **Proceed without a key pair**
5. On the **Configure Security Group** step, create a new security group allowing:
   - SSH traffic
   - HTTPS traffic
   - HTTP traffic
6. Review and launch the instance

## Step 2: Connect to the instance

Navigate to your EC2 instance, click **Connect**, leave the default parameters, and click **Connect** again.

## Step 3: Set up your instance

<Info>
  The following commands install a Node.js agent. Adapt them to your specific stack if needed.
</Info>

```bash
# Update the instance
sudo yum update -y

# Install Git
sudo yum install git -y

# Clone your repo
git clone your-repo-link

# Install Node.js and npm
sudo yum install npm -y

# Install dependencies
cd your-repo-directory
npm install
```

Set up all environment variables provided by the Forest Admin environment creation wizard, and add the `APPLICATION_PORT` variable:

```bash
export APPLICATION_PORT=3310
```

Start the agent:

```bash
npm run start:watch
```

## Step 4: Adjust security group rules

1. Navigate to your EC2 instance's security group
2. Click **Edit inbound rules**
3. Add a Custom TCP inbound rule to allow port `3310`

## Step 5: Create a target group

1. In EC2, navigate to **Target Groups → Create Target Groups**
2. Ensure target type is **Instance**
3. Choose HTTP to port `3310`
4. Ensure VPC matches your EC2 instance
5. Set health checks to `/forest`
6. Select your instance and click **Include as pending below**
7. Create the target group

## Step 6: Request an SSL certificate (ACM)

1. Navigate to **ACM → Request a certificate**
2. Enter your domain name and validate via DNS validation
3. After the certificate is created, click **Create records in Route 53**
4. Wait for the certificate to be validated (usually under 1 minute)

## Step 7: Set up an Application Load Balancer

1. In EC2, navigate to **Load Balancers → Create Load Balancer**
2. Choose **Application Load Balancer**
3. Ensure the ALB is in the same VPC as your EC2 instance
4. Select all regions
5. Remove the default security group and select the one associated with your instance
6. Add an **HTTPS listener** pointing to your target group and certificate
7. After creating the ALB, copy the **DNS name**

## Step 8: Add CNAME to Route53

1. Navigate to **Route53** and choose your hosted zone
2. Create a `CNAME` record with:
   - Name: your domain (as used in the ACM certificate)
   - Value: the ALB's DNS name

## Step 9: Finalize

Check your domain — you should be able to access your Forest Admin agent. 🎉

<Info>
  This is a basic setup. For production-ready deployments, consider using RDS for the database, tightening security groups, and enabling auto-scaling. Refer to the [AWS documentation](https://docs.aws.amazon.com/index.html) for more.
</Info>

## Next step

Once your agent is deployed and accessible via HTTPS, go back to Forest Admin and [create your production environment](/guides/deployment/production#step-2-create-the-production-environment-in-forest-admin).

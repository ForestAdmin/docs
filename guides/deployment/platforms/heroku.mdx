This guide walks you through deploying your Forest Admin agent to Heroku using Heroku Git.

## Prerequisites

[Sign up for Heroku](https://signup.heroku.com/) if you don't have an account yet.

## Step 1: Create a Heroku application

1. Log in to Heroku
2. Click **New → Create new app**
3. Enter an app name and choose a region

## Step 2: Deploy using Heroku Git

Follow the Heroku guide **"Deploy using Heroku Git"** shown in your app's Deploy tab, then push your agent code:

```bash
git push heroku master
```

Heroku will automatically detect your stack (Node.js, Ruby, etc.) and build your application.

<Tip>
  Your agent is now deployed to Heroku. 🎉
</Tip>

## Step 3: Set environment variables

In your Heroku app's **Settings → Config Vars**, add:

| Variable | Value |
|---|---|
| `FOREST_ENV_SECRET` | Your Forest Admin production env secret |
| `FOREST_AUTH_SECRET` | Your Forest Admin auth secret |
| `DATABASE_URL` | Your production database connection string |
| `NODE_ENV` | `production` (Node.js) |

## Next step

Once your agent is deployed and accessible via HTTPS (Heroku provides SSL automatically on `*.herokuapp.com`), go back to Forest Admin and [create your production environment](/guides/deployment/production#step-2-create-the-production-environment-in-forest-admin).

<Warning>
  Deploying to Heroku does not automatically set up your Forest Admin production environment. You still need to configure it in the Forest Admin UI.
</Warning>

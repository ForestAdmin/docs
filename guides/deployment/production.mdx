## Prerequisites

- A working Forest Admin agent running locally or in staging
- A production server accessible via HTTPS
- A production database

## Step 1: Deploy your agent to a server

Before creating the production environment in Forest Admin, deploy your agent code to your production server. See the platform-specific guides:

<CardGroup cols={2}>
  <Card title="Deploy to AWS" icon="aws" href="/guides/deployment/platforms/aws">
    Deploy to an EC2 instance with load balancer
  </Card>
  <Card title="Deploy to Heroku" icon="bolt" href="/guides/deployment/platforms/heroku">
    Deploy with Heroku Git
  </Card>
</CardGroup>

Make sure to set the following environment variables on your server:

```bash
FOREST_ENV_SECRET=<your_production_env_secret>
FOREST_AUTH_SECRET=<your_production_auth_secret>
DATABASE_URL=<your_production_database_url>
NODE_ENV=production   # Node.js
# or
RAILS_ENV=production  # Ruby
```

<Warning>
  Your agent must be accessible via **HTTPS**. The URL must not end with a trailing `/`.
</Warning>

## Step 2: Create the Production Environment in Forest Admin

Click **"Deploy to production"** in the top banner of your Forest Admin project, or go to **Project Settings → Environments**.

### Enter your agent URL

Input the URL of your production server — this is the URL where your agent is running.

### Connect your database

Enter your production database credentials. These are used only to generate environment variables and are never stored.

### Set environment variables

Follow the on-screen instructions to add the generated environment variables to your server. Once your server is detected and running with the correct variables, a **"Finish"** button will appear.

## Step 3: Deploy layout changes

Once your production environment is created, use the Forest Admin CLI to deploy your layout changes from development to production:

```bash
forest deploy
```

<Danger>
  Deploy with care — this action cannot be reverted. Make sure to deploy your backend code changes **before** running `forest deploy`.
</Danger>

<Info>
  Since all environments' layouts depend on your reference (production) environment, `forest deploy` applies layout changes to all your project environments.
</Info>

## Checklist

Before going live:

- [ ] Agent deployed and accessible via HTTPS
- [ ] `FOREST_ENV_SECRET` and `FOREST_AUTH_SECRET` set on the server
- [ ] Production environment created in Forest Admin UI
- [ ] Health check returns 200: `curl https://your-agent.com/forest`
- [ ] All collections visible in the UI
- [ ] CRUD operations work correctly

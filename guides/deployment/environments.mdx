## Overview

Each Forest Admin project has multiple environments — at minimum a **Development** environment (created automatically) and a **Production** environment. You can also create **Remote** environments for staging, QA, pre-production, etc.

Each environment has:
- Its own agent URL (pointing to your deployed server)
- Its own database credentials
- Its own layout (UI configuration)
- Its own `FOREST_ENV_SECRET`

## Creating a Production Environment

See [Production Deployment](/guides/deployment/production) for the full guide.

## Creating a Remote Environment

Remote environments are useful for staging, QA, and other intermediate environments.

1. Go to **Project Settings → Environments**
2. Click **"Add a new environment"**
3. Enter a name (e.g. "Staging")
4. Enter the agent URL for that environment
5. Enter the database credentials for that environment
6. Follow the on-screen instructions to set the generated environment variables on your server

<Info>
  You can create a Remote Environment before going to production — the order is up to you.
</Info>

## Environment hierarchy

Environments have an **origin** — the environment their layout is based on. By default:

- Remote environments have Production as their origin
- Development branches can have any environment as their origin

You can change the origin of an environment to create complex workflows:

```
dev → staging → preprod → production
```

To change the origin of an environment, click on it in Project Settings and use the **"Set Origin"** section.

<Warning>
  All child environments will be refreshed based on the new hierarchy.
</Warning>

## Setting an environment as Production

If you need to promote a Remote environment to Production:

1. Click on the environment in Project Settings
2. Click **"Set as production"**

<Warning>
  The previous production environment will take the new production as its origin. All children layouts will be refreshed. Layout changes that cannot be applied will be ignored.
</Warning>

<Info>
  The environment you want to set as production must have the current reference (production) as its origin.
</Info>

## Deleting an environment

From the environment's detail page in Project Settings, you can delete an environment.

<Warning>
  Deleting an environment is irreversible.
</Warning>

## Environment variables

Each environment requires these variables on the agent server:

| Variable | Description |
|---|---|
| `FOREST_ENV_SECRET` | Unique secret for this environment (provided by Forest Admin) |
| `FOREST_AUTH_SECRET` | Secret for signing JWT tokens (shared across environments) |
| `DATABASE_URL` | Connection string for this environment's database |

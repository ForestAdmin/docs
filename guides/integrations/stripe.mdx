There are two ways to integrate Stripe with Forest:

- **Stripe as a datasource** — surface Stripe customers, subscriptions, payments, and invoices as collections you can browse and search alongside your own data. Best when you want operators to navigate Stripe data inside Forest.
- **Stripe in Smart Actions** — call the Stripe API from action handlers to charge, refund, update subscriptions, send invoices. Best when Stripe data lives elsewhere (e.g. you replicate it to your own database) and you only need to *act* on Stripe.

You can use both together.

## Connect Stripe as a datasource

The Stripe datasource is a Node.js-only experimental package.

<Note>
  Available for Node.js agents. The datasource is part of [Forest experimental](/guides/forestadmin-experimental) — community-maintained but production-ready for the included resources.
</Note>

### Install

```bash
npm install @forestadmin-experimental/datasource-stripe
```

### Connect

```javascript
import { createAgent } from '@forestadmin/agent';
import { createStripeDataSource } from '@forestadmin-experimental/datasource-stripe';

const agent = createAgent({
  authSecret: process.env.FOREST_AUTH_SECRET,
  envSecret: process.env.FOREST_ENV_SECRET,
  isProduction: process.env.NODE_ENV === 'production',
});

agent.addDataSource(
  createStripeDataSource({
    secretKey: process.env.STRIPE_SECRET_KEY,
  })
);
```

Once connected, Stripe collections (Customers, Subscriptions, Payments, Invoices, etc.) appear in Forest alongside your own data. Operators can browse, search, and filter Stripe records.

### Source code

The datasource is open source — see the [`forestadmin-experimental` repository](https://github.com/ForestAdmin/forestadmin-experimental) for the latest list of supported Stripe resources.

## Trigger Stripe operations from Smart Actions

For operations that change Stripe state (refunds, subscription updates, invoice generation), build Smart Actions that call the Stripe SDK.

<Tabs>
  <Tab title="Refund a payment">
    ```javascript
    import Stripe from 'stripe';
    const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);

    agent.customizeCollection('orders', orders => {
      orders.addAction('Refund', {
        scope: 'Single',
        form: [
          { label: 'Amount (cents)', type: 'Number', isRequired: true },
          { label: 'Reason', type: 'Enum', enumValues: ['duplicate', 'fraudulent', 'requested_by_customer'] },
        ],
        execute: async (context, resultBuilder) => {
          const order = await context.getRecord(['stripePaymentIntentId']);
          const { Amount, Reason } = context.formValues;

          try {
            const refund = await stripe.refunds.create({
              payment_intent: order.stripePaymentIntentId,
              amount: Amount,
              reason: Reason,
            });
            return resultBuilder.success(`Refund created: ${refund.id}`);
          } catch (err) {
            return resultBuilder.error(`Refund failed: ${err.message}`);
          }
        },
      });
    });
    ```
  </Tab>
  <Tab title="Cancel a subscription">
    ```javascript
    agent.customizeCollection('users', users => {
      users.addAction('Cancel subscription', {
        scope: 'Single',
        execute: async (context, resultBuilder) => {
          const user = await context.getRecord(['stripeSubscriptionId']);

          if (!user.stripeSubscriptionId) {
            return resultBuilder.error('No active subscription on this user.');
          }

          await stripe.subscriptions.cancel(user.stripeSubscriptionId);
          return resultBuilder.success('Subscription cancelled');
        },
      });
    });
    ```
  </Tab>
  <Tab title="Send an invoice">
    ```javascript
    agent.customizeCollection('invoices', invoices => {
      invoices.addAction('Send invoice', {
        scope: 'Single',
        execute: async (context, resultBuilder) => {
          const invoice = await context.getRecord(['stripeInvoiceId']);
          await stripe.invoices.sendInvoice(invoice.stripeInvoiceId);
          return resultBuilder.success('Invoice sent');
        },
      });
    });
    ```
  </Tab>
</Tabs>

## Display Stripe data on your own records

If you store Stripe references on your own records (e.g. `User.stripeCustomerId`), enrich your records with computed fields that pull live data from Stripe:

```javascript
agent.customizeCollection('users', users => {
  users.addField('stripeBalance', {
    columnType: 'Number',
    dependencies: ['stripeCustomerId'],
    getValues: async records => {
      return Promise.all(records.map(async r => {
        if (!r.stripeCustomerId) return null;
        const customer = await stripe.customers.retrieve(r.stripeCustomerId);
        return customer.balance;
      }));
    },
  });
});
```

For performance, batch the API calls or cache results — the `getValues` function receives all records on the page at once, so you can use `stripe.customers.list({ ids: [...] })` patterns where supported.

## Webhooks from Stripe

Forest agents are Express (Node.js) or Rails (Ruby) apps. Mount your Stripe webhook handler alongside your agent:

```javascript
import express from 'express';
import Stripe from 'stripe';

const app = express();
const stripe = new Stripe(process.env.STRIPE_SECRET_KEY);

// Mount Stripe webhook BEFORE the agent
app.post('/stripe/webhook', express.raw({ type: 'application/json' }), (req, res) => {
  const event = stripe.webhooks.constructEvent(
    req.body,
    req.headers['stripe-signature'],
    process.env.STRIPE_WEBHOOK_SECRET,
  );

  // Process the event — e.g. update your database
  // Forest will reflect changes on the next read

  res.json({ received: true });
});

await agent.mountOnExpress(app).start();
app.listen(3310);
```

## Best practices

<AccordionGroup>
  <Accordion title="Use test mode in development">
    Set `STRIPE_SECRET_KEY` to a test mode key (`sk_test_...`) in development environments. Forest's environment system makes this easy — different `STRIPE_SECRET_KEY` values per environment.
  </Accordion>
  <Accordion title="Handle rate limits">
    Stripe's API rate-limits at 100 requests/sec by default. If a computed field calls Stripe per record on a page of 25 records, that's fine — but be aware of the limit when building bulk actions over thousands of records.
  </Accordion>
  <Accordion title="Audit destructive actions">
    Refunds, cancellations, and payment captures are sensitive. Use [approval workflows](/product/collaborate/approval-workflows) to require sign-off before these actions execute, and scope them to specific roles in Project Settings → Roles.
  </Accordion>
  <Accordion title="Don't store API keys in code">
    Always read `STRIPE_SECRET_KEY` from environment variables. The agent's environment system handles this per environment.
  </Accordion>
</AccordionGroup>

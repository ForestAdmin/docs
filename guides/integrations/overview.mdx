Forest integrates with external services in three different ways depending on what you want to do:

- **Connect external data** — surface data from a third-party API as a collection in Forest (Stripe customers, HubSpot companies, Airtable records, etc.). This is a [datasource](/get-started/connect/data-sources/overview).
- **Trigger external services from actions** — call a webhook, send a Slack message, charge a Stripe customer. This is a [Smart Action](/product/process/actions/overview) or a [no-code action](/product/process/actions/nocode-actions).
- **Authenticate via your identity provider** — Single Sign-On with SAML 2.0, SCIM provisioning. See [Authentication](/get-started/control/authentication/overview).

This guide covers the most common third-party integrations in the Forest ecosystem.

## Integration patterns

<AccordionGroup>
  <Accordion title="When to use a datasource vs a Smart Action">
    Use a **datasource** when you want to *browse, filter, and search* external records as if they were a database collection. Use a **Smart Action** when you want to *trigger a one-shot operation* against the external API.

    Example: HubSpot contacts as a browsable list → datasource. "Send this contact a welcome email via HubSpot" → Smart Action.
  </Accordion>
  <Accordion title="Webhooks (incoming)">
    Forest agents are Express (Node.js) or Rails (Ruby) applications, so you can mount your own routes alongside the Forest routes. Configure the third party to POST to your custom route and process the webhook there. Trigger Forest updates by writing to your database — the Forest UI will reflect changes on the next read.
  </Accordion>
  <Accordion title="Webhooks (outgoing)">
    Use a no-code action with the **Webhook** type, or build a Smart Action that calls the external service. See [No-code actions](/product/process/actions/nocode-actions).
  </Accordion>
  <Accordion title="API key management">
    Store third-party API keys in environment variables on your agent server, never in code. The agent reads them at runtime via `process.env` (Node.js) or `ENV` (Ruby).
  </Accordion>
</AccordionGroup>

## Data privacy

When a user logs into the Forest UI, their browser connects to:

1. **Forest servers** — to retrieve the layout configuration (UI settings)
2. **Your agent** — to retrieve your data

<Tip>
  Your data is transmitted directly between your agent and the user's browser. **It never transits through Forest servers.**
</Tip>

## Authentication tokens

Requests are protected using two separate JSON Web Tokens:

| Token | Purpose |
|---|---|
| `FOREST_ENV_SECRET` | Authenticates requests to Forest servers. Provided by Forest — unique per environment. |
| `FOREST_AUTH_SECRET` | Authenticates requests to your agent. Chosen freely by you — unknown to Forest. |

<Warning>
  Both secrets must never be disclosed. The `FOREST_AUTH_SECRET` is only known to you — Forest cannot help you recover it.
</Warning>

The JWT payload contains the requesting user's details, available on any authenticated request to your agent:

```json
{
  "id": "172",
  "email": "user@example.com",
  "firstName": "Jane",
  "lastName": "Doe",
  "team": "Operations",
  "role": "Manager",
  "tags": [{ "key": "country", "value": "FR" }],
  "renderingId": "4998"
}
```

You can use this information in your agent to implement custom behaviors (e.g., scoping data by team, auditing actions).

## IP whitelisting

The IP whitelisting feature lets you restrict access to both the Forest UI and your agent to a specific list of trusted IP addresses or ranges.

Configure it in **Project Settings → Security → IP Whitelisting**.

## DMZ & VPN

You can host your agent in any cloud architecture compliant with your security requirements — DMZ, VPN, private network, etc. Forest only needs to reach your agent URL to retrieve the schema; all data requests come directly from the user's browser.

## Third-party tracking

Forest provides an option to disable all third-party vendors that could track metadata from your users' browsers.

<Info>
  This feature is available on the Pro plan and above.
</Info>

## Self-hosted security checklist

- Use HTTPS — your agent URL must use the `https://` protocol
- Use a separate `FOREST_ENV_SECRET` for each environment (development, staging, production)
- Store secrets in your platform's secret manager (AWS Secrets Manager, Heroku Config Vars, etc.)
- Never commit `.env` files to version control
- Restrict your agent's network access to only what's needed (database, external APIs)
- Keep your agent and its dependencies updated

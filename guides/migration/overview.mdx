If you're running a legacy agent — `forest-express-sequelize`, `forest-express-mongoose`, `forest-rails`, or `django-forestadmin` v1 — this section walks you through migrating to the current agent generation.

The current generation is the foundation of everything Forest has shipped since the original admin-panel era — workflows, MCP server for AI agents, multi-datasource, decision traces. Legacy agents only expose the original admin-panel capabilities.

<Note>
  Most teams complete migration in 1-2 days. You can run v1 and v2 in parallel with zero downtime.
</Note>

## Why migrate

| | Legacy agents | Current agents |
| --- | --- | --- |
| Workflows orchestration | ❌ | ✅ |

| TypeScript | ❌ | ✅ (Node.js) |
| Active development | ❌ | ✅ |
| New features | ❌ | ✅ |
| Performance | OK | Optimized query layer |

## Migration approach

<Steps>
  <Step title="Install the new agent alongside your current one">
    The new agent runs on a different port. Both can run simultaneously without conflict, so you can compare behavior side by side.
  </Step>
  <Step title="Configure datasources">
    Re-declare your data sources using the new datasource API. See [Datasources](/guides/migration/from-v1/steps/datasources).
  </Step>
  <Step title="Migrate customizations one feature at a time">
    Smart Actions, Smart Fields, Smart Relationships, Smart Segments. Each has its own step page.
  </Step>
  <Step title="Test in parallel">
    Run both agents and compare results before flipping traffic.
  </Step>
  <Step title="Switch and clean up">
    Point your Forest environment to the new agent's URL. Once stable, remove the legacy packages.
  </Step>
</Steps>

## Run v1 and v2 in parallel

Run both agents on different ports during migration:

```bash
# Terminal 1 — legacy agent on port 3000
npm run start:v1

# Terminal 2 — new agent on port 3001
npm run start:v2
```

Switch your Forest environment URL when you're confident the new agent is correct.

## FAQ

<AccordionGroup>
  <Accordion title="Do I need to migrate everything at once?">
    No. Migrate incrementally. Run v1 and v2 in parallel and switch when ready. Most teams migrate one feature at a time over a few days.
  </Accordion>
  <Accordion title="Will my UI break during migration?">
    No. Your Forest UI configuration (layouts, segments, dashboards) is preserved. The agent change is transparent to end users.
  </Accordion>
  <Accordion title="Can I revert if something goes wrong?">
    Yes. Keep the legacy agent running in parallel until you're confident. You can switch back instantly by pointing your environment URL back at the legacy agent.
  </Accordion>
  <Accordion title="What about my data?">
    Your database is not affected. Only the agent code changes — the database schema, records, and credentials stay exactly as they are.
  </Accordion>
  <Accordion title="Are there breaking changes?">
    The customization API is different, but the concepts are the same. Most changes are syntax updates. The step-by-step pages cover every conversion side by side.
  </Accordion>
</AccordionGroup>

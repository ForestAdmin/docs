## Prerequisites

- Node.js 18+ installed
- Your database URI ready
- A Forest Admin account ([sign up](https://app.forestadmin.com/signup) if needed)

## Steps

<Steps>
  <Step title="Create a new project">
    Go to [app.forestadmin.com](https://app.forestadmin.com), create a new project and choose **Self-Hosted**.
  </Step>

  <Step title="Follow the onboarding">
    The Forest Admin UI will guide you through the setup. It will ask for your database URI and generate a ready-to-run application with your environment variables pre-configured.
  </Step>

  <Step title="Start your agent">
    In the generated project directory, run:

    ```bash
    npm start
    ```

    You should see:
    ```
    [Forest] 🌳  Your agent is running at http://localhost:3310
    ```
  </Step>

  <Step title="Open your panel">
    Forest Admin automatically opens your panel in the browser. Your database schema has been detected — collections, relationships, and CRUD operations are ready.
  </Step>
</Steps>

<Warning>
Never commit your `.env` file to version control. Add it to `.gitignore`.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Agent not starting" icon="circle-xmark">
    - Check that your `.env` file is present and loaded
    - Ensure port `3310` is not already in use
    - Run `curl http://localhost:3310/forest` — should return Forest Admin metadata
  </Accordion>

  <Accordion title="No tables showing up" icon="table">
    - Verify your `DATABASE_URL` is correct and the database is reachable
    - Check that the database user has read permissions on tables
  </Accordion>
</AccordionGroup>
